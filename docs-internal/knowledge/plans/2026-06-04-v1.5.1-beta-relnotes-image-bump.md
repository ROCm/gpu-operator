# v1.5.1-beta Release Notes

- **Date:** 2026-06-04
- **Author:** praveen
- **Related PR(s):** TBD

## Context

`docs/releasenotes.md` has no entry for v1.5.1 (beta). This PR adds the release notes
section only; image tag and version bumps across `docs/` are deferred to the GA release.

## Approach

Insert new top-level section `## GPU Operator v1.5.1 (beta) Release Notes` above v1.5.0
in `docs/releasenotes.md`.

**Release Highlights:**
- MI350P: K8s + OpenShift, ROCm 7.13+ / amdgpu 6.19+. Full feature set (KMM, device plugin, node labeller, exporter, test runner, DCM, DRA, NPD). ANR not fully supported.
- Radeon AI Pro (RDNA 4, e.g. R9700S): K8s only, ROCm 7.13+. Supported: driver mgmt, device plugin, metrics. Not supported: DCM partitioning, XGMI scheduling, full ECC metrics.
- Exporter: `--exit-on-rocpctl-error` and `--exit-on-agent-down` Helm chart args.

**Platform Support:** K8s 1.34, 1.35 + OCP 4.21 (unchanged from v1.5.0).

**Known Limitations:**
- ANR not fully validated on MI350P and Radeon AI.
- Node labeller `amd.com/gpu.product-name` = `AMD_Radeon_Graphics` on Radeon AI instead of specific model name (e.g. `AMD Radeon AI Pro R9700S`) — open bug.
- CPER-based health detection and `GPU_AFID_ERRORS` disabled in exporter `v1.5.1-beta.0` for **all platforms** (including MI2xx/MI3xx). Users depending on these should not upgrade the exporter.
- `gfx_activity` spurious idle uptick on Radeon AI (cosmetic).

## Scope

- **In scope:** `docs/releasenotes.md` only.
- **Out of scope:** Image tags, helm `--version=` flags, `docs/conf.py`, and all other `docs/` files — deferred to GA.
